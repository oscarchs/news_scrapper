Google lidia con un problema creciente en el mercado de aplicaciones, y lo hace ejerciendo presión en un punto de dolor inusual para el desarrollador: el Play Store. Desde este 1 de noviembre, los servicios de Google comenzarán a limitar la presencia de aplicaciones que no añadan herramientas recientes de Android.De acuerdo con Krish Vitaldevara, director de gestión de productos en Google, este es un proceso continuo que apunta a mejorar la seguridad de las apps en todo el ecosistema Android.“Actualmente requerimos nuevas aplicaciones y actualizaciones para apuntar a un nivel de API de Android dentro de un año del lanzamiento de la última versión principal del sistema operativo Android. Las aplicaciones nuevas y las actualizaciones de aplicaciones que no cumplan con este requisito no se pueden publicar en Google Play”, señaló el ejecutivo.Android y los niveles de APIEl nivel de API es parte del desarrollo de las plataformas de Android, y son parte de una identificación que añade el nombre y el número de versión. Por ejemplo, la actual versión de Android 12 (número) “Snow Cone” (nombre) tiene un nivel de API 31 y 32, debido a que también cuenta con la versión 12L.La estrategia de Google es limitar la aparición de aplicaciones por debajo de dos años de una actualización mayor de Android. Para las apps nuevas, el límite será de un año.“La razón detrás de esto es simple”, describe Vitaldevara. “Los usuarios con los últimos dispositivos o aquellos que están completamente al día con las actualizaciones esperan aprovechar todo el potencial de las protecciones de privacidad y seguridad que Android tiene para ofrecer. Ampliar nuestros requisitos de API de nivel de destino protegerá a los usuarios de instalar aplicaciones más antiguas que pueden no tener estas protecciones implementadas”.Google limitará apps en AndroidSin embargo, hay dos condiciones que pueden ser consideradas positivas. Por un lado, las apps que ya hemos descargado previamente y no actualizaron el nivel de API podrán ser descargadas nuevamente, en caso necesitemos hacerlo. Por otro lado, la gran mayoría de apps en Google Play ya cuenta con un nivel de API estable.Esto último se debe a que Google ha dinamizado sus políticas de publicación para los desarrolladores que solicitan revisión, y ahora se exige que las apps deban tener, al menos, un nivel de API que tenga hasta un año tras la última versión importante del sistema operativo.Con esta medida, las aplicaciones abandonadas por desarrolladores comenzarán a desaparecer de las sugerencias para nuevos usuarios. Además, limitará el uso de alguna API de menor nivel para evadir políticas actualizadas de seguridad y privacidad.
